Installation Guide

Guppty is a small indentation-based programming language written in Rust. Install Rust, clone the repository, install the local command, then run any `.gup` program from your terminal.

Quick Installation

For macOS, Linux, and Windows.

Rust + Cargo
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
git clone https://github.com/adrian-1-cardona/Guppty.git
cd Guppty
cargo install --path .
guppty examples/program.gup

rustc --version

Clone the repository

Download the project source from GitHub and move into the repo folder.

git clone https://github.com/adrian-1-cardona/Guppty.git
3

Install the local CLI

This builds the binary and installs `guppty` into Cargo's bin directory.

cargo install --path .
4

Run an example

guppty examples/program.gup

Terminal

Guppty CLI

The CLI expects a `.gup` file. By default it lexes, parses, compiles, and runs the program on the stack-based VM. Pass `--interp` to use the older tree-walking interpreter.

Run with the VM

Best default for normal use.

guppty examples/hello.gup

Run with the interpreter

Useful for comparing behavior while developing the language.

guppty examples/hello.gup --interp

Run without installing

Good while editing the Rust project itself.

cargo run -- examples/program.gup

Purpose

Why Guppty?

Guppty is small enough to read from top to bottom, but complete enough to teach the real shape of a language implementation: tokens, parsing, AST nodes, compilation, bytecode, runtime values, scopes, functions, closures, and error reporting.

The project is also designed to be changed. Keyword syntax lives in `src/syntax.rs`, examples double as behavior checks, and the VM is separated from the legacy interpreter so changes can be tested without hiding how the language works.

Language

Syntax

Indentation based

Variables, values, and output

Use `out(...)` to print values. Numbers, floats, strings, chars, booleans, and arrays are supported.

x = 5
name = "Guppty"
ready = true
letter = 'g'
out(name)
out(x + 10)

If, while, and for

Blocks are controlled by indentation. `range(1 through 4)` includes the start and walks toward the end.

score = 85
if score >= 80
    out("You passed!")
else
    out("Keep studying!")

for i in range(1 through 3)
    out(i)

Functions and returns

Define a function with `name(args)` followed by an indented body.

add(a, b)
    return a + b

out(add(2, 3))

Closures

Functions can capture values from the scope where they were defined.

makeAdder(x)
    addIt(y)
        return x + y
    return addIt

adder = makeAdder(5)
out(adder(3))

Comments

Double slash comments can sit on their own line or after code.

// this is a comment
out(42) // this is an inline comment

Programs

Examples

The `examples/` folder is the best place to learn the language by reading small programs. Matching expected output files live in `examples/expected/` and are checked by the test suite.

Hello World

out("Hello World!")
guppty examples/hello.gup

Recursion

factorial(n)
    if n <= 1
        return 1
    return n * factorial(n - 1)

out(factorial(5))
guppty examples/recursion_factorial.gup

Closure Counter

makeCounter(start)
    count = start
    increment()
        count = count + 1
        return count
    return increment
guppty examples/closure_counter.gup

Notes

What I Learned

Language features connect fast. A small syntax change can touch lexing, parsing, compilation, runtime values, examples, and tests.

Good examples are tests with a friendly face. The `.gup` files show users what to write and protect behavior at the same time.

Errors matter. Reading a source file, rendering parse failures, and showing runtime errors clearly makes the CLI feel much more real.

Community

Contribute

Small, focused pull requests are easiest to review. If you change language behavior, update or add a `.gup` example and the matching expected output.

Troubleshooting

Common Issues

`cargo` or `rustc` is not found

Install Rust from rustup.rs, then restart your terminal so Cargo's bin directory is on your PATH.

`guppty` is not found after install

Run `cargo install --path .` from the repository root. If it succeeds but the command is missing, add Cargo's bin directory to your shell PATH.

The CLI cannot read my file

Check that the file path is correct and that the file ends in `.gup`. The command shape is `guppty path/to/file.gup`.

Indented blocks behave unexpectedly

Keep the block body consistently indented after `if`, `else`, `while`, `for`, and function definitions.

How do I check that everything still works?

Run `cargo test`. The test suite builds the project and checks the example programs against expected output files.

Publishing

GitHub Pages

This documentation site is intentionally built in `/docs`. To publish it on GitHub Pages, open the repository settings and choose the `/docs` folder as the Pages source for the branch you publish from.

  1. Push this repository to GitHub.
  2. Open Settings, then Pages.
  3. Set Source to Deploy from a branch.
  4. Choose the branch, usually `main`, and select the /docs folder.
  5. Save. GitHub will serve `docs/index.html` as the public documentation page.

For this repository, the expected project-site URL is `https://adrian-1-cardona.github.io/Guppty/` after Pages is enabled.
