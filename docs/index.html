Installation Guide

Guppty is a small, indentation-based programming language written entirely in Rust. It was designed from the ground up as a learning tool, something you can install in a few minutes and immediately start experimenting with. The language compiles your source files into bytecode and runs them on a lightweight virtual machine, which means you get to see every stage of the pipeline from raw text to actual execution without needing any external dependencies beyond the Rust toolchain itself.

The installation process is intentionally straightforward. There is no package manager to configure, no complex build system to fight with, and no hidden environment variables to set. You install Rust, you clone the repository, you run a single Cargo command, and then you have a working command-line tool that can execute any file ending in the .gup extension. The whole thing takes about two minutes on a modern internet connection, and at the end of it you will have a fully functional programming language sitting on your machine, ready to run.

Quick Installation

For macOS, Linux, and Windows

Rust + Cargo

The installation steps below will guide you through setting up Guppty on your system regardless of which operating system you are running. The process is nearly identical across all three platforms because Rust and Cargo handle the cross-platform compilation details for you. The only real difference is how you initially install the Rust toolchain itself. On macOS and Linux, you use the official rustup installer script. On Windows, you can use winget or download the installer directly from the Rust website. Once Rust is in place, every other step is exactly the same no matter what machine you are sitting in front of.

  1. First, you need to install the Rust programming language on your Mac. If you have never used Rust before, this command downloads the official rustup installer, which sets up the Rust compiler, the Cargo package manager, and all the standard tools you need. The script is safe to run and will ask you to confirm before making any changes to your system. Once it finishes, you will need to restart your terminal or run the source command it suggests so that the new commands become available in your current shell session.

    curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

  2. Next, download the entire Guppty source code from GitHub. This command creates a new folder called Guppty in whatever directory you are currently in, and it pulls down every file in the project including the source code, the example programs, the test suite, and this documentation site. You will need git installed on your machine for this to work, but most Macs come with git preinstalled through the Xcode command line tools.

    git clone https://github.com/adrian-1-cardona/Guppty.git

  3. Move into the project folder that was just created by the clone command. Everything from this point forward needs to happen inside this directory because Cargo looks for the Cargo.toml file to understand how to build the project.

    cd Guppty

  4. Now install the guppty command-line tool. This command tells Cargo to compile the project in release mode and place the resulting binary into your Cargo bin directory, which is typically located at ~/.cargo/bin. After this completes, you will be able to type guppty from anywhere on your system and it will work, because Cargo's bin directory should already be on your PATH from when you installed Rust in the first step.

    cargo install --path .

  5. Finally, run one of the included example programs to confirm that everything is working correctly. The program.gup file is a small demonstration that exercises several language features at once. If you see output printed to your terminal without any error messages, your installation is complete and Guppty is ready to use.

    guppty examples/program.gup

  1. Start by installing Rust on your Linux machine. The rustup installer works on virtually every distribution and will set up the compiler, Cargo, and all the standard Rust tooling. If you are on a system that does not have curl installed, you can install it first through your distribution's package manager. After the installer finishes, restart your shell or source the environment file it points you to.

    curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

  2. Clone the Guppty repository from GitHub. This pulls down the entire project including all source files, example programs, expected output files, and the documentation. You will need git installed, which most Linux distributions include by default or make available through their package manager.

    git clone https://github.com/adrian-1-cardona/Guppty.git

  3. Change into the newly created project directory. All the build commands expect to be run from the root of the repository where the Cargo.toml configuration file lives.

    cd Guppty

  4. Install the guppty binary from this local checkout. Cargo compiles the project and places the resulting executable in ~/.cargo/bin, which should already be on your PATH from the Rust installation. The compilation will take a moment the first time as it downloads and compiles all dependencies.

    cargo install --path .

  5. Run the example program to verify that your installation is working. If text appears in your terminal without errors, everything is set up correctly and you can start writing your own .gup files.

    guppty examples/program.gup

  1. Install the Rust toolchain on Windows using winget, which is the built-in package manager available on Windows 10 and later. This command downloads and installs rustup, which in turn installs the Rust compiler and Cargo. After the installation completes, you need to close and reopen your terminal so that the new PATH entries take effect. If you do not have winget available, you can alternatively download the rustup installer directly from rustup.rs.

    winget install Rustlang.Rustup

  2. Clone the Guppty repository. This works the same way on Windows as it does on any other platform. You will need git installed, which you can get through Git for Windows or through winget. The command creates a Guppty folder in your current directory containing all the project files.

    git clone https://github.com/adrian-1-cardona/Guppty.git

  3. Navigate into the project folder. On Windows you can use either forward slashes or backslashes for this command, both work fine in modern terminals like PowerShell or Windows Terminal.

    cd Guppty

  4. Install the guppty command from the local source code. Cargo compiles everything and places the executable in your Cargo bin directory, which is usually located at C:\Users\YourName\.cargo\bin. This directory should already be on your system PATH from the Rust installation step.

    cargo install --path .

  5. Test the installation by running one of the included example programs. Note that on Windows you use backslashes for file paths. If the program produces output without any error messages, you are all set and ready to start using Guppty.

    guppty .\examples\program.gup

First Run

Quick Start

Install Rust

If you have already completed the installation steps above, this section gives you a condensed overview of what you just did and explains the reasoning behind each step in a bit more detail. Think of it as the quick-reference version that you can come back to later when you want to remember the workflow without reading through all the explanations again. The four steps below represent the complete lifecycle of getting Guppty from nothing to running on your machine.

The first thing to understand is that Guppty is not distributed as a prebuilt binary yet. You are compiling it yourself from source code, which means you need the Rust compiler and its build tool, Cargo. This is actually a benefit because it means you can look at and modify any part of the language whenever you want. You are not just a user of Guppty, you have the entire implementation sitting right there on your machine, fully transparent and ready to be explored or changed.

1

Install the Rust toolchain

Guppty requires the Rust 2021 edition compiler. If the command below prints a version number, you already have Rust installed and can skip ahead to step two. If it says the command is not found, head to rustup.rs and follow the installation instructions for your operating system. The whole process takes about a minute and gives you everything you need to compile Rust programs.

rustc --version
2

Clone the repository

Pull down the complete Guppty source code from GitHub. This gives you the compiler, the virtual machine, the interpreter, all the example programs, the test suite, and this documentation. Cargo needs the Cargo.toml file at the root of this repository to understand how to build the project, so make sure you are inside the Guppty directory before running any build commands.

git clone https://github.com/adrian-1-cardona/Guppty.git
3

Install the local CLI

Run this inside the Guppty folder. It compiles the project in release mode and places the guppty binary into your Cargo bin directory, which should already be on your system PATH. After this step, you can run guppty from any directory on your machine. If you ever want to update to a newer version, just pull the latest changes from GitHub and run this command again.

cargo install --path .
4

Run an example

The default execution path compiles your .gup file into bytecode and runs it on Guppty's stack-based virtual machine. The examples folder contains dozens of small programs that demonstrate different language features, from simple hello-world printing to recursive functions and closures. Try running a few of them to get a feel for how the language works before writing your own programs.

guppty examples/program.gup

That said, there are times when you might want the source code without going through git. Terminal

Guppty CLI

The Guppty command-line interface is designed to be as simple as possible. You give it a file, and it runs that file. There are no subcommands to memorize, no configuration files to write, and no flags you need to remember for everyday use. The entire interface boils down to one pattern: you type guppty followed by the path to a .gup file, and the language does the rest. It lexes the source text into tokens, parses those tokens into an abstract syntax tree, compiles the tree into bytecode instructions, and then executes those instructions on the stack-based virtual machine. All of that happens in milliseconds, and the only thing you see is whatever output your program produces.

There is one optional flag worth knowing about. If you pass --interp after the file path, Guppty will skip the compiler and virtual machine entirely and instead run your program using the older tree-walking interpreter. This interpreter reads the AST directly and evaluates it node by node, which is a fundamentally different execution strategy than compiling to bytecode first. The results should be identical for any correct program, but the interpreter is useful when you are debugging language behavior or when you want to compare how the two backends handle a particular edge case. It is also a nice educational tool because it shows you a second way that programming languages can execute code.

If you are actively developing Guppty itself and do not want to reinstall the binary every time you make a change, you can use cargo run to execute programs directly from the source tree. This compiles and runs the project in one step, which is faster during development because you skip the install step. The double dash tells Cargo that everything after it is an argument for the program rather than an argument for Cargo itself.

Run with the VM

This is the standard execution path that you will use the vast majority of the time. Your source code gets compiled into bytecode and executed on the virtual machine. It is fast, predictable, and produces clean output.

guppty examples/hello.gup

Run with the interpreter

The tree-walking interpreter is the alternative backend that evaluates your AST directly without compiling to bytecode first. It is useful for comparison, debugging, and understanding how two different execution models can produce the same results from the same source code.

guppty examples/hello.gup --interp

Run without installing

When you are working on the Guppty source code itself and making frequent changes, this command compiles and runs in one step without needing to reinstall the binary each time. The double dash separates Cargo's own arguments from the arguments that get passed through to the Guppty program.

cargo run -- examples/program.gup

Project

About Guppty

Guppty is a small indentation-based programming language for learning how source code becomes something a computer can run. It starts with a `.gup` file, turns that text into tokens, builds an AST, compiles the AST into bytecode, and runs the bytecode on a virtual machine.

The project keeps the older tree-walking interpreter available with `--interp` because it is useful for comparison. The VM is the default path, and the interpreter gives the project a second way to check that language behavior still makes sense.

benchmarks

VM vs tree-walker on recursion_fib.gup

VM 2.69 ms
Tree-walker 2.64 ms

These numbers are simple end-to-end CLI averages from 200 release-build runs of `target/release/guppty examples/recursion_fib.gup`. The example is tiny, so process startup is a big part of the time and the two paths are very close. The point of this benchmark is to keep both backends measurable and to make future performance changes easy to compare.

Purpose

Why Guppty?

Guppty is tiny on purpose. You can read the whole thing, change a keyword, add a small feature, and actually see how that choice moves through the language pipeline.

It is not trying to be the next giant production language. It is a clear little playground for learning lexers, parsers, bytecode, scopes, functions, closures, and the very satisfying moment when a file named `hello.gup` finally runs.

Language

Syntax

Indentation based

Variables, values, and output

Use `out(...)` to print values. Numbers, floats, strings, chars, booleans, and arrays are supported.

x = 5
name = "Guppty"
ready = true
letter = 'g'
out(name)
out(x + 10)

If, while, and for

Blocks are controlled by indentation. `range(1 through 4)` includes the start and walks toward the end.

score = 85
if score >= 80
    out("You passed!")
else
    out("Keep studying!")

for i in range(1 through 3)
    out(i)

Functions and returns

Define a function with `name(args)` followed by an indented body.

add(a, b)
    return a + b

out(add(2, 3))

Closures

Functions can capture values from the scope where they were defined.

makeAdder(x)
    addIt(y)
        return x + y
    return addIt

adder = makeAdder(5)
out(adder(3))

Comments

Double slash comments can sit on their own line or after code.

// this is a comment
out(42) // this is an inline comment

Programs

Examples

The `examples/` folder is the easiest way to get a feel for the language. Each program is small enough to skim, and the matching `examples/expected/` file shows exactly what should print.

Hello World

out("Hello World!")
guppty examples/hello.gup

Recursion

factorial(n)
    if n <= 1
        return 1
    return n * factorial(n - 1)

out(factorial(5))
guppty examples/recursion_factorial.gup

Closure Counter

makeCounter(start)
    count = start
    increment()
        count = count + 1
        return count
    return increment
guppty examples/closure_counter.gup

Under the Hood

Technologies Used

Guppty uses a small set of plain technologies so each part of the language pipeline is easy to inspect. Rust provides the command-line program, Cargo runs builds and tests, the compiler creates bytecode, and the VM runs that bytecode in a predictable stack-based model.

Source.gup file
Lexertokens
ParserAST
Compilerbytecode
VMstdout

Rust 2021

Rust was used because it is a good fit for a language tool: it builds a fast local CLI, has strong types for tokens and AST nodes, and ships with Cargo for repeatable builds and tests.

Stack VM

The stack VM is the default backend because it separates compilation from execution. That makes the runtime easier to inspect, benchmark, and improve without changing the parser every time.

Tree Interpreter

The tree-walking interpreter stays available behind `--interp` because it runs the AST directly. It is useful when checking whether a compiler or VM change still matches the simpler execution path.

Integration Tests

The tests run real `.gup` examples and compare them with expected stdout. That keeps the documentation examples and the language behavior tied together.

Notes

What I Learned

Language features connect fast. A small syntax change can touch lexing, parsing, compilation, runtime values, examples, and tests.

Examples are quiet little guardrails. The `.gup` files show people what to write and keep the implementation honest at the same time.

Errors matter more than they look. A friendly message with a real location makes the CLI feel like a tool instead of a guessing game.

Community

Contribute

Small, focused pull requests are easiest to review. If you change language behavior, update or add a `.gup` example and the matching expected output.

Troubleshooting

Common Issues

`cargo` or `rustc` is not found

Install Rust from rustup.rs, then restart your terminal so Cargo's bin directory is on your PATH.

`guppty` is not found after install

Run `cargo install --path .` from the repository root. If it succeeds but the command is missing, add Cargo's bin directory to your shell PATH.

The CLI cannot read my file

Check that the file path is correct and that the file ends in `.gup`. The command shape is `guppty path/to/file.gup`.

Indented blocks behave unexpectedly

Keep the block body consistently indented after `if`, `else`, `while`, `for`, and function definitions.

How do I check that everything still works?

Run `cargo test`. The test suite builds the project and checks the example programs against expected output files.

Publishing

GitHub Pages

This documentation site is intentionally built in `/docs`. To publish it on GitHub Pages, open the repository settings and choose the `/docs` folder as the Pages source for the branch you publish from.

  1. Push this repository to GitHub.
  2. Open Settings, then Pages.
  3. Set Source to Deploy from a branch.
  4. Choose the branch, usually `main`, and select the /docs folder.
  5. Save. GitHub will serve `docs/index.html` as the public documentation page.

For this repository, the expected project-site URL is `https://adrian-1-cardona.github.io/Guppty/` after Pages is enabled.
